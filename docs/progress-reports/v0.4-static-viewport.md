# Static Viewport Release Review

**Review date:** 20 June 2026

## Starting Point

Version 0.3.0 added sound, haptics, confetti, PWA metadata, a horizontal roster carousel and mobile-safe controls. Browser measurement found that team selection fit the viewport, but tournament exceeded a 1280 x 720 desktop by 28 px. Battle measured 1,240 px tall on desktop and 2,228 px on mobile, with sticky controls covering arena content.

## Completed

- Locked application screens to the dynamic viewport with no document scrolling.
- Replaced the scrollable roster strip with a button-controlled three-card desktop gallery and one-card mobile focus.
- Kept all five tournament stages aligned on desktop and introduced one-stage mobile focus navigation.
- Replaced full stat cards in battle with aligned 4:3 artwork panels.
- Added click-to-inspect full-card modals for both active fighters.
- Kept team status, health, battle messages and all five actions visible at once.
- Compacted the tutorial into a non-scrolling two-by-two mobile modal.
- Preserved full artwork loading and resilient image fallbacks on every screen.

## Browser Evidence

- Desktop team: three aligned cards at 280 x 381 within 1280 x 720.
- Desktop tournament: five aligned stages within 1280 x 720.
- Desktop battle: two 430 x 323 portraits and all controls within 1280 x 720.
- Mobile team, tournament, battle, tutorial and card detail modal fit within 390 x 844.
- All tested pages reported document dimensions equal to viewport dimensions.
- No broken character artwork was found.

## Status

**GREEN — static desktop and mobile game flow complete.**

## Next Improvements

- Add optional swipe gestures to the focus galleries while keeping arrow controls.
- Add a compact match-history modal using the saved battle record.
- Add deterministic end-to-end browser tests for every screen size.

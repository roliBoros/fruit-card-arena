# Fruit Card Arena — Master Plan

## Goal

Turn the original hand-drawn collection into a professional card set and a browser-based single-player game for ages 10+.

## Phase 0 — Catalogue and setup

- Record every original character and visible statistic.
- Mark unclear names or numbers rather than guessing.
- Establish the repository, documentation and technical scaffold.
- Create a simple username-only local profile.

**Gate:** repository works as a coherent project and all uncertain source data is clearly flagged.

## Phase 1 — Visual system and production prototypes

- Lock the approved visual direction.
- Produce individual high-resolution versions of Dragon-Fruit, Ava, Meloni and the Bonus card.
- Create a reusable blank frame and card back.
- Keep text and numbers separate from generated artwork wherever possible.

**Gate:** four individual cards pass readability, consistency and age-appropriateness checks.

## Phase 2 — Complete card set

- Produce the remaining character artwork.
- Add controlled names, statistics, rarity and special abilities.
- Export web PNG/WebP files and print-ready versions.
- Create a contact sheet and complete collection PDF.

**Gate:** every card passes spelling, statistics, consistency and export checks.

## Phase 3 — Game interface

- Username entry and local profile.
- Main menu, collection and team selection.
- Battle arena, results screen and tournament route.
- Responsive desktop, tablet and mobile layouts.

**Gate:** a player can enter a name, choose a team and reach a complete battle screen.

## Phase 4 — Battle engine

- Three-card teams.
- Attack, Guard and character Special actions.
- One +20 Bonus card per battle.
- AI decisions, health, victory and defeat rules.
- Card movement, impact flash, shake and floating damage feedback.

**Gate:** a full match can be played without refreshing or editing data.

## Phase 5 — Testing and polish

- Balance all cards.
- Add difficulty choices and a short tutorial.
- Test mobile and desktop browsers.
- Add reduced-motion support and loading optimisation.
- Deploy a shareable build.

**Gate:** no known gameplay-blocking errors.

## Phase-review format

Every phase ends with:

1. Planned work
2. Completed work
3. Evidence
4. Incomplete or changed items
5. Problems discovered
6. Green, yellow or red status
7. Exact next action
